# Screenshot capture guide

Screenshots are optional portfolio assets and are not generated by the build.
Use fictional candidate data only. Never capture real resumes, private history,
passwords, tokens, phone numbers, addresses, or local filesystem paths.

## Desktop checklist

Use a `1440 × 900` browser viewport, 100% zoom, consistent light from the same
browser theme, and crop only after the page is fully loaded.

| Filename | App state | Show |
| --- | --- | --- |
| `01-login-page.png` | Signed-out login tab | Login panel and validation-safe empty state. |
| `02-register-page.png` | Signed-out register tab | Registration form with fictional account data. |
| `03-analyze-resume.png` | Authenticated, Analyze mode | Upload form and fictional job description. |
| `04-analysis-results.png` | Completed Analyze request | ATS, similarity, skills, and result dashboard. |
| `05-role-recommendations.png` | Roles mode with a fictional resume | Ranked roles and explainable score details. |
| `06-resume-improvement.png` | Improve mode | Priority actions and expandable feedback. |
| `07-analysis-history.png` | Authenticated with saved fictional reports | History list and ownership-safe actions. |
| `08-developer-branding.png` | Bottom of authenticated dashboard | Meet the Developer section and footer. |
| `09-api-documentation.png` | Backend at `/docs` | Swagger route groups without secrets. |
| `10-docker-status.png` | Optional local Compose run | Service names/status only; remove environment values. |

Before saving, remove visible email accounts, tokens, passwords, personal
resumes, local paths, database details, and browser extensions. Use PNG, keep
text readable, avoid excessive empty space, and optimize file size.

## Mobile checklist

Repeat the most important captures at `390 × 844`: login, upload form, analysis
result, role recommendations, feedback, history, and branding/footer. Confirm
there is no horizontal overflow and that buttons, headings, and expandable
sections remain readable.

## Adding captures

Save approved images in `docs/images/application/` using the filenames above.
Only add Markdown image references after the corresponding file exists. Review
every image for privacy before committing it.
